<task type="auto">
  <name>Task 1: Create mini player BrowserWindow module and vite config</name>
  <files>src/main/mini-player.ts, electron.vite.config.ts</files>
  <action>
  **src/main/mini-player.ts:**
  Create a `createMiniPlayer()` factory function that returns a BrowserWindow. Per D-01, D-03, D-04, D-05, D-16:

  - Frameless (`frame: false`), `alwaysOnTop: true` with level `'floating'`
  - Default size 320×100, min 280×80, max 500×150
  - `backgroundColor: '#0f0f0f'` matching main window
  - `show: false` (starts hidden, shown when toggled)
  - Same preload script as main window: `join(__dirname, '../preload/index.mjs')`
  - `contextIsolation: true`, `sandbox: true`, `nodeIntegration: false`
  - `skipTaskbar: true` — mini player shouldn't appear as separate taskbar entry
  - `resizable: true`

  Position/size persistence via electron-store (KD-05):
  - On create: read `miniPlayer.bounds` from store. If exists, use those bounds. If not, position at bottom-right of primary display (`screen.getPrimaryDisplay().workAreaSize`).
  - On `move` and `resize` events: save current bounds `{ x, y, width, height }` to store under `miniPlayer.bounds`.
  - Import `screen` from electron for display info.

  Load the mini player HTML:
  - Dev: `process.env['ELECTRON_RENDERER_URL'] + '/mini-player.html'`
  - Production: `join(__dirname, '../renderer/mini-player.html')`

  Export: `createMiniPlayer()` returning `BrowserWindow`.

  **electron.vite.config.ts:**
  Add multi-page renderer configuration. Import `resolve` from `path`. Update renderer config:
  ```typescript
  renderer: {
    build: {
      rollupOptions: {
        input: {
          index: resolve(__dirname, 'src/renderer/index.html'),
          'mini-player': resolve(__dirname, 'src/renderer/mini-player.html')
        }
      }
    }
  }
  ```

  **src/renderer/mini-player.html:**
  Per D-06, D-10, D-17, D-18. Structure:
  ```
  #mini-player
    .mini-drag         (thin drag area at top, ~12px height)
      .mini-controls   (expand + close buttons, positioned right in drag area)
    .mini-content      (main content area)
      #album-art       (img element, 60×60, with music-note fallback)
      .track-info
        #track-title   (primary text, truncate ellipsis)
        #track-artist  (secondary text, dimmer, truncate ellipsis)
      .playback-controls
        #btn-prev      (previous button)
        #btn-play      (play/pause button)
        #btn-next      (next button)
  ```

  CSP meta tag: `default-src 'self'; style-src 'self' 'unsafe-inline'; script-src 'self'; img-src 'self' https://*.googleusercontent.com https://*.ytimg.com https://lh3.googleusercontent.com` (album art comes from Google's CDN).

  Link to `./styles/mini-player.css`. Script: `./mini-player.ts` as module.

  **src/renderer/styles/mini-player.css:**
  Per D-15, D-16. Reuse the same CSS variable palette from titlebar.css (copy the `:root` block).
  - `#mini-player`: full viewport, dark bg, flex column
  - `.mini-drag`: height 12px, `-webkit-app-region: drag`, flex with justify-end
  - `.mini-controls` buttons: small (20×12px), no-drag, same hover pattern as titlebar
  - `.mini-content`: flex row, `align-items: center`, padding 8px, gap 10px, flex 1
  - `#album-art`: 60×60px, `border-radius: 4px`, `object-fit: cover`, flex-shrink 0. Fallback: show a music note SVG as background when img has no src.
  - `.track-info`: flex column, min-width 0 (for text truncation), flex 1
  - `#track-title`: font-size 13px, font-weight 600, white, single-line truncate (`white-space: nowrap; overflow: hidden; text-overflow: ellipsis`)
  - `#track-artist`: font-size 11px, `rgba(255,255,255,0.6)`, single-line truncate
  - `.playback-controls`: flex row, gap 4px, flex-shrink 0
  - Playback buttons: 32×32px, same hover/active states as titlebar buttons
  - Expand button: chevron-up SVG (mirrors the chevron-down in main title bar)
  - Close button: X SVG, red hover like titlebar close

  **src/renderer/mini-player.ts:**
  - Subscribe to `window.electronAPI.onMetadataUpdate()` — update album art src, title text, artist text. When metadata is null, show "Not playing" / clear art.
  - Wire playback buttons: `#btn-play` → `electronAPI.playPause()`, `#btn-prev` → `electronAPI.previous()`, `#btn-next` → `electronAPI.next()`
  - Update play/pause button icon based on `metadata.isPlaying` (show pause icon when playing, play icon when paused)
  - Wire `#btn-expand` → `electronAPI.expandFromMini()` (new IPC channel, wired in Plan 02)
  - Wire `#btn-close` → `electronAPI.close()`
  - Use inline SVG for all icons (established pattern from Phase 1)
  - Handle album art load error: hide img, show fallback music note
