# Phase 1 — UI Design Contract

> Visual and interaction contract for the custom title bar and frameless window chrome.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none |
| Icon library | inline SVG (4 icons: close, maximize, restore, minimize, shrink-to-mini chevron) |
| Font | `-apple-system, system-ui, "Segoe UI", Roboto, sans-serif` (system font stack — native Windows feel) |

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-icon gaps within button groups |
| sm | 8px | Button internal padding (horizontal) |
| md | 16px | Title bar horizontal padding (left/right edges) |
| lg | 24px | Not used in this phase |
| xl | 32px | Title bar height (matches D-05: ~32px) |

Exceptions: Title bar height is fixed at 32px (D-05). Window control buttons are 46px wide × 32px tall (standard Windows 11 caption button dimensions).

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 13px | 400 (regular) | 1.0 |
| Label | 12px | 400 (regular) | 1.0 |

Note: D-02 specifies **no title text** in the title bar. Typography is minimal — only used if a tooltip or accessible label is rendered. The two sizes above cover tooltip text (Body) and any future micro-label (Label). No Heading or Display sizes needed for this phase.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0f0f0f` | Title bar background, seamless with YT Music's main dark background |
| Secondary (30%) | `#212121` | Button hover background (minimize, maximize, shrink-to-mini) |
| Accent (10%) | `#ffffff` | Window control button icons (close ×, maximize □, minimize ─, chevron ▾) |
| Destructive | `#c42b1c` | Close button hover background only |

### Extended Palette

| Token | Value | Usage |
|-------|-------|-------|
| `--titlebar-bg` | `#0f0f0f` | Title bar background (matches YT Music dark theme body) |
| `--titlebar-btn-hover` | `#2d2d2d` | Hover state for minimize, maximize, shrink-to-mini buttons |
| `--titlebar-btn-active` | `#3a3a3a` | Active/pressed state for non-destructive buttons |
| `--titlebar-close-hover` | `#c42b1c` | Close button hover background (Windows 11 convention) |
| `--titlebar-close-active` | `#b22a1a` | Close button active/pressed background |
| `--titlebar-icon` | `#ffffff` | Icon color for all button icons (default state) |
| `--titlebar-icon-close-hover` | `#ffffff` | Close icon remains white on red hover background |
| `--titlebar-text` | `#ffffff` | Primary text (tooltips only — no visible title text per D-02) |
| `--titlebar-text-muted` | `#aaaaaa` | Secondary/disabled text |
| `--titlebar-border` | `transparent` | No border between title bar and content (D-03) |

Accent reserved for: window control button icons only (close, maximize, minimize, shrink-to-mini). No other accent usage in this phase.

### Color Rationale
- `#0f0f0f` is YouTube Music's primary dark theme background color, ensuring the title bar is visually seamless with the embedded content (D-01, D-03).
- `#212121` / `#2d2d2d` are YouTube Music's elevated surface colors, used for hover feedback (D-07).
- `#c42b1c` is the standard Windows 11 close-button red (D-07: "red highlight on close button").

## Component Inventory

### Title Bar (`<div id="titlebar">`)

| Property | Value | Source |
|----------|-------|--------|
| Height | 32px | D-05 |
| Background | `#0f0f0f` | D-01 |
| Border bottom | none / transparent | D-03 |
| Layout | flexbox, `justify-content: space-between` | D-04 (left group + right group) |
| `-webkit-app-region` | `drag` on the bar, `no-drag` on buttons | D-08 |

### Left Button Group

| Element | Icon | Size | Behavior |
|---------|------|------|----------|
| Shrink-to-mini | Chevron-down SVG (▾) | 46px × 32px | Click → IPC to main process (wired in Phase 3). D-06 |

### Right Button Group (standard Windows order)

| Element | Icon | Size | Hover BG | Behavior |
|---------|------|------|----------|----------|
| Minimize | Horizontal line (─) | 46px × 32px | `#2d2d2d` | `ipcRenderer.send('window-minimize')` |
| Maximize/Restore | Square (□) / Overlapping squares (⧉) | 46px × 32px | `#2d2d2d` | `ipcRenderer.send('window-maximize')`. Icon toggles between maximize and restore states. |
| Close | Cross (×) | 46px × 32px | `#c42b1c` | `ipcRenderer.send('window-close')` |

### Icon Specifications

All icons are inline `<svg>` elements, 10px × 10px viewBox, `stroke: currentColor`, `stroke-width: 1px`, `fill: none`.

| Icon | Description | SVG Path Concept |
|------|-------------|-----------------|
| Minimize | Single horizontal line, centered | `M 0 5 L 10 5` |
| Maximize | Square outline | `M 0 0 L 10 0 L 10 10 L 0 10 Z` |
| Restore | Two overlapping squares (back square offset 2px up-right) | Front: `M 0 2 L 8 2 L 8 10 L 0 10 Z` Back: `M 2 0 L 10 0 L 10 8` |
| Close | Two diagonal lines forming × | `M 0 0 L 10 10 M 10 0 L 0 10` |
| Shrink-to-mini | Downward chevron | `M 2 3 L 5 7 L 8 3` |

### Drag Region

| Property | Value | Source |
|----------|-------|--------|
| Area | Entire title bar excluding button groups | D-08, D-10 |
| Double-click | Toggle maximize/restore | D-08 |
| CSS | `-webkit-app-region: drag` on titlebar container, `-webkit-app-region: no-drag` on all `<button>` elements | Electron docs |

## Accessibility

- All buttons have `aria-label` attributes matching tooltip copy
- `title` attributes for native tooltips
- `tabindex` ordering: left-to-right (mini → minimize → maximize → close)
- `:focus-visible` styling for keyboard navigation
- `@media (prefers-reduced-motion: reduce)` — disable transitions
- `@media (forced-colors: active)` — Windows High Contrast support
- SVG icons: `pointer-events: none` to prevent stealing clicks from buttons
