<objective>
Extract now-playing metadata from YouTube Music's DOM via executeJavaScript polling and expose playback controls (play/pause, next, previous) via IPC. Main process acts as the metadata hub — polls ytmView, diffs changes, broadcasts to subscribed renderers.

Purpose: This is the data foundation for Phase 3 (mini player). Without metadata extraction and playback controls, the mini player has nothing to display and no way to control playback.

Output: Working metadata pipeline (ytmView → main → renderer) and playback control path (renderer → main → ytmView), all wired through IPC.
</objective>

**Create `src/main/ytm-selectors.ts`** with centralized DOM selectors for YouTube Music's player bar. Per D-10 and D-11, prefer aria-labels and data attributes where available.

The executor MUST inspect the live YouTube Music page (https://music.youtube.com) to discover current selectors. Key areas to inspect:

- **Player bar** — the bottom bar that shows current track info. Look for `ytmusic-player-bar` or similar custom element.
- **Track title** — typically in a `yt-formatted-string` with class like `title` inside the player bar.
- **Artist name** — typically a `yt-formatted-string` or `span` with class like `subtitle` or `byline` in the player bar.
- **Album art** — `img` element inside the player bar, look for `image` class or the thumbnail `img` within a song-image container.
- **Play/pause button** — look for `#play-pause-button` or a button with `aria-label` "Play"/"Pause" inside the player bar.
- **Next/previous buttons** — buttons near play/pause with `aria-label` "Next"/"Previous".
- **Progress** — look for `#progress-bar` or a slider element. Duration/progress text often in time display elements.
- **Shuffle/repeat** — buttons with `aria-label` "Shuffle"/"Repeat", often with an `aria-pressed` or status attribute.

Export as:
```typescript

  playerBar: '...',
  title: '...',
  artist: '...',
  albumArt: '...',
  playPauseBtn: '...',
  nextBtn: '...',
  prevBtn: '...',
  progressBar: '...',
  timeInfo: '...',
  shuffleBtn: '...',
  repeatBtn: '...'
} as const
```

Use CSS selector strings. If a selector can't be discovered, leave a `// TODO: verify selector` comment and use the best guess.

**Create `src/main/metadata.ts`** — the metadata scraping engine.

Per D-01 and D-02, this module:
1. Accepts `ytmView.webContents` and `mainWindow.webContents` references
2. Runs a `setInterval` (every ~1 second) that calls `ytmView.webContents.executeJavaScript()` with a script that reads DOM selectors from `YTM_SELECTORS` and returns a `NowPlayingMetadata` object
3. Diffs the new metadata against the previous state (per D-08 — only broadcast on actual change). Use JSON.stringify comparison for simplicity.
4. On change, calls `mainWindow.webContents.send('metadata-update', metadata)` to broadcast to the title bar renderer (and future mini player)
5. Exports `startMetadataPolling(ytmView, mainWindow)` and `stopMetadataPolling()`

The `executeJavaScript` script must be a self-contained string (no imports — it runs in the YTM page context). It should:
- Query each selector
- Extract text content / attribute values
- Parse time strings (e.g., "3:45" → 225 seconds)
- Return a plain object matching NowPlayingMetadata shape
- Wrap in try/catch — return null on any error (per D-12, graceful degradation)

Per D-12: If `executeJavaScript` throws or returns null, emit `metadata-update` with null metadata (don't crash, don't log spam — maybe log once per failure streak).

**Modify `src/main/window.ts`** — export `ytmView` reference so metadata module can access it. The `createMainWindow()` already returns `{ mainWindow, ytmView }` so this is just about ensuring the references are passed correctly.

**Modify `src/main/index.ts`** — after `createMainWindow()`, import and call `startMetadataPolling(ytmView, mainWindow)`. Start polling AFTER `ytmView.webContents` fires `'did-finish-load'` on YouTube Music (not immediately — DOM isn't ready yet).

```typescript
// In index.ts, after createMainWindow():
ytmView.webContents.on('did-finish-load', () => {
  startMetadataPolling(ytmView, mainWindow)
})
```

Also stop polling on `window-all-closed`.
