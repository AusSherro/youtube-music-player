# Milestone v1.0: YouTube Music Desktop Player

**Status:** ✅ SHIPPED 2026-04-24
**Phases:** 01–04
**Total Plans:** 7

## Overview

First release of YTP — a dedicated Windows desktop wrapper for YouTube Music with an always-on-top mini player. Wraps YouTube Music in a frameless Electron window with Google sign-in, Widevine DRM, metadata extraction, playback controls, global media keys, and a fully featured mini player.

## Phases

### Phase 1: Core Wrapper & Window Chrome

**Goal**: Electron app that loads YouTube Music in a frameless desktop window with Google sign-in support, Widevine DRM, and a custom dark title bar.
**Requirements:** [WRAP-01, WRAP-02, WRAP-03, WNDW-01]
**Plans**: 2 plans

Plans:
- [x] 01-01-PLAN.md — Scaffold Electron project with WebContentsView wrapper
- [x] 01-02-PLAN.md — Custom dark title bar with window controls

**Details:**
- WebContentsView (not deprecated BrowserView) for YouTube Music embedding
- UA spoofing strips Electron token for Google sign-in compatibility
- Widevine DRM via Electron 41 built-in support
- Preload bridge with contextBridge.exposeInMainWorld pattern
- Window module pattern: window.ts exports createMainWindow()
- Custom dark title bar with minimize/maximize/close/shrink-to-mini buttons
- Inline SVG icons, CSS custom properties for theming

**Success Criteria:** App launches, loads YouTube Music, sign-in works, audio plays, title bar controls function.

### Phase 2: Metadata Extraction & Playback Controls

**Goal**: Extract now-playing metadata from YouTube Music DOM and provide playback controls via IPC. Main process polls ytmView with executeJavaScript, diffs changes, broadcasts to renderers. Playback commands simulate DOM clicks.
**Requirements:** [META-01, META-02, CTRL-01]
**Plans**: 1 plan

Plans:
- [x] 02-01-PLAN.md — Metadata extraction pipeline + playback controls + preload API

**Details:**
- NowPlayingMetadata type and PlaybackCommand type in shared/types.ts
- YTM DOM selectors centralized in ytm-selectors.ts
- 1-second polling interval with JSON.stringify diffing for change detection
- Graceful degradation: emit null metadata on scraper failure
- Playback controls execute via DOM click simulation

### Phase 3: Mini Player

**Goal**: Resizable always-on-top mini player showing now-playing info and playback controls.
**Requirements:** [MINI-01, MINI-02, MINI-03, MINI-04]
**Plans**: 2 plans

Plans:
- [x] 03-01-PLAN.md — Create mini player window module and renderer (HTML/CSS/TS)
- [x] 03-02-PLAN.md — Wire toggle mechanism, metadata broadcast, and app integration

**Details:**
- BrowserWindow with always-on-top (floating level), frameless, position persistence via electron-store
- Album art, track info, playback controls, dark-theme styling
- Toggle mechanism between main window and mini player
- Dual metadata broadcast to both windows
- Mini player close triggers app.quit()

**Success Criteria:** Mini player appears on shrink click, shows live metadata, playback controls work, expand returns to full window, position persists.

### Phase 4: Media Keys & Polish

**Goal**: Global media key support, mini player progress bar with seeking, app icon, startup loading state, and lifecycle edge case handling.
**Requirements:** [MKEY-01, PROG-01, ICON-01, SPLS-01, PLSH-01, PLSH-02]
**Plans**: 2 plans

Plans:
- [x] 04-01-PLAN.md — Media keys, seek IPC, lifecycle edge cases (reload, navigation)
- [x] 04-02-PLAN.md — Progress bar, splash screen, app icon, navigation-away UI

**Details:**
- globalShortcut registers MediaPlayPause, MediaNextTrack, MediaPreviousTrack
- Seek-to-position via video.currentTime through executeJavaScript
- Navigation-away detection broadcasts to all renderers with overlay UI
- Reload recovery: stopMetadataPolling before restart
- Seekable progress bar (3px→5px on hover)
- Splash screen with spinner, hidden on ytm-loaded IPC
- Custom app icon (red circle with white play triangle)

**Success Criteria:** Media keys control playback globally, progress bar shows and seeks, splash screen on startup, navigation-away handled, reload recovery works.

## Milestone Summary

**Key Decisions:**
- KD-01: Electron 41+ with WebContentsView (modern replacement for deprecated BrowserView)
- KD-02: electron-vite for build tooling (fast HMR, separate main/preload/renderer targets)
- KD-03: electron-builder for packaging (best Windows NSIS/portable support)
- KD-04: Vanilla TS + CSS for UI (framework overkill for ~50 lines of UI)
- KD-05: electron-store for persistence (JSON file in userData)
- executeJavaScript polling from main process (not MutationObserver) for metadata
- UA spoofing for Google sign-in compatibility
- DOM click simulation for playback controls

**Issues Resolved:**
- Google sign-in blocking resolved via UA spoofing
- Preload CJS format, album art selector, base64 proxy fixes
- Destroyed window guards on metadata broadcast
- Splash screen visibility (defer ytmView child add until did-finish-load)
- Window icon set on BrowserWindow for dev mode visibility

- None
